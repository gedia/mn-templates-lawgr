= metanorma-lawgr: Πρότυπο Ελληνικού Νομοθετήματος

Αυτό το πρότυπο δημιουργεί ένα σκελετό εργασίας για τη σύνταξη ελληνικού
νομοθετικού κειμένου με τη χρήση του metanorma-lawgr.

== Δομή αρχείων

[source]
----
document.adoc            ← Κύριο αρχείο (metadata + includes)
metanorma.yml            ← Ρυθμίσεις metanorma
sections/
  00-preamble.adoc       ← Προοίμιο
  01-section.adoc        ← Πρώτη ενότητα κυρίου σώματος
  02-section.adoc        ← Δεύτερη ενότητα
  03-section.adoc        ← Τρίτη ενότητα
  ...                      (προσθέστε/αφαιρέστε όσες χρειάζεστε)
  99-colophon.adoc       ← Κλείσιμο (δημοσίευση, υπογραφές)
----

== Πώς να οργανώσετε τα αρχεία

Κάθε αρχείο στον φάκελο `sections/` αντιπροσωπεύει **μία ενότητα στο ίδιο
ιεραρχικό επίπεδο** του νόμου.  Η επιλογή του επιπέδου εξαρτάται αποκλειστικά
από τη δομή του συγκεκριμένου νομοθετήματος και γίνεται ελεύθερα από τον συντάκτη του:

- Αν π.χ. ο νόμος χωρίζεται μόνο σε **Μέρη** (Μέρος Α, Β, Γ…), κάθε αρχείο είναι ένα Μέρος.
- Αν π.χ. ο νόμος χωρίζεται μόνο σε **Βιβλία**  (ΒΙΒΛΙΟ ΠΡΩΤΟ, ΔΕΥΤΕΡΟ…), κάθε αρχείο είναι ένα Βιβλίο.
- Αν χωρίζεται σε περισσότερα επίπεδα (π.χ. έχει και **Μέρη** και **Κεφάλαια** ή και κάποια άλλη υποδιαίρεση), επιλέξτε ελεύθερα ένα μόνο επίπεδο για την αντιστοίχιση με τα αρχεία.
- Κάθε υποδιαίρεση κάτω από το επίπεδο που επιλέξατε για τα αρχεία θα είναι μέρος αυτού του αρχείου.
** π.χ. αν επιλέξατε τα αρχεία στον φάκελο sections να αντιπροσωπεύουν **Μέρη**, τότε όλα τα πιθανά **Τμήματα**, **Κεφάλαια** και **Άρθρα** του κάθε Μέρους θα πρέπει να περιλαμβάνονται στο αρχείο του Μέρους που αντιστοιχούν.
- Αν είναι σύντομος και χωρίζεται μόνο σε **Άρθρα**, κάθε αρχείο μπορεί να ομαδοποιεί σχετικά άρθρα, ή να χρησιμοποιήσετε ένα μόνο αρχείο για όλα τα Άρθρα.

Τα αρχεία `00-preamble.adoc` και `99-colophon.adoc` είναι ειδικά:

- **`00-preamble.adoc`** — Το εισαγωγικό κείμενο πριν από το κύριο σώμα
  (π.χ. «Έχοντας υπόψη…», «αποφασίζει:»).
- **`99-colophon.adoc`** — Το τελικό κείμενο μετά το τελευταίο άρθρο
  (π.χ. εντολή δημοσίευσης στο ΦΕΚ, στοιχεία υπογραφόντων).

Η αρίθμηση στο όνομα του αρχείου (01-, 02-, 03-…) καθορίζει μόνο τη σειρά
εμφάνισης μέσω των `include::` στο `document.adoc`, το οποίο θα πρέπει να 
ενημερώσετε με τις μεταβολές σας.  Προσθέστε, αφαιρέστε ή μετονομάστε αρχεία ελεύθερα.

== Δημιουργία νέου εγγράφου

Για τη δημιουργία νέου εγγράφου από αυτό το πρότυπο, χρησιμοποιήστε την εντολή
`metanorma new` δίνοντας τη διαδρομή προς τον φάκελο `templates/` του gem:

[source,sh]
----
# Νόμος / Κανονισμός (προεπιλογή doctype: act)
metanorma new my-law --type lawgr --doctype act \
  --template /path/to/metanorma-lawgr/templates

# Προεδρικό Διάταγμα
metanorma new my-pd --type lawgr --doctype pd \
  --template /path/to/metanorma-lawgr/templates

# Απόφαση
metanorma new my-apofasi --type lawgr --doctype ap \
  --template /path/to/metanorma-lawgr/templates
----

Αντικαταστήστε `/path/to/metanorma-lawgr` με την πραγματική διαδρομή του gem
στο σύστημά σας.  Η εντολή θα δημιουργήσει έναν νέο φάκελο (π.χ. `my-law/`)
με αντίγραφο όλων των αρχείων του πρότυπου.

== Οδηγίες μεταγλώττισης σε HTML

[source,sh]
----
metanorma compile document.adoc -t lawgr -x html
----

== Ιεραρχία δομικών στοιχείων

Η ελληνική νομοθεσία χρησιμοποιεί τα ακόλουθα δομικά στοιχεία.
Όλα είναι προαιρετικά — ο συντάκτης επιλέγει ποια χρειάζεται
ανάλογα με το νομοθέτημα.

|===
| Στοιχείο | AsciiDoc | Αρίθμηση | Σημείωση

| **Βιβλίο**     | `[heading=book]`       | ΒΙΒΛΙΟ ΠΡΩΤΟ, ΔΕΥΤΕΡΟ | Σπάνιο, μόνο σε πολύ μεγάλους νόμους
| **Μέρος**      | `[heading=part]`       | ΜΕΡΟΣ Α, Β, Γ         | Συνήθης διαίρεση
| **Τμήμα**      | `[heading=tmima]`      | ΤΜΗΜΑ Α, Β             |
| **Κεφάλαιο**   | `[heading=chapter]`    | ΚΕΦΑΛΑΙΟ Α, Β          |
| **Άρθρο**      | `[heading=article]`    | Άρθρο 1, 2, 3          | Βασική μονάδα — σχεδόν πάντα παρόν
| **Υποάρθρο**   | `[heading=subarticle]` | 1, 2, 3                | Μη κανονιστικό — το βρίσκουμε για ομαδοποίηση άρθρων κάτω από κοινό τίτλο, με συμμετοχή στην αρίθμιση π.χ. ΑΔΑΕ 304/2025 - Άρθρο 3: Διακυβέρνηση Πολιτικής Ασφάλειας και Διαχείριση Κινδύνου - **3.1. Πολιτική Ασφάλειας Δικτύων και Υπηρεσιών**
| **Παράγραφος** | `[heading=paragraph]`  | 1, 2, 3                | Mπορεί να είναι ένθετο επ' αόριστον (indefinite nesting)
| **Custom**     | `[heading=custom]`     | Α, Β, Γ                | Μη κανονιστικό — το βρίσκουμε για ομαδοποίηση άρθρων κάτω από κοινό τίτλο, χωρίς να συμμετέχει στην αρίθμιση π.χ. ΑΔΑΕ 304/2025 - 5.1. Φυσική Πρόσβαση και Περιβαλλοντική Ασφάλεια **Α. Φυσική Πρόσβαση**
|===

Οι αριθμημένες λίστες (περιπτώσεις) χρησιμοποιούν ελληνικά αριθμητικά:
α), β), γ)… στ), ζ)… ι), ια)…

== Τύποι εγγράφων (doctype)

- `act` — Νόμος / Κανονισμός (προεπιλογή)
- `pd` — Προεδρικό Διάταγμα
- `ap` — Απόφαση
- `egk` — Εγκύκλιος
- `other` — Άλλο
